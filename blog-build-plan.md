## 总体节奏

两条主线并行推进，互不阻塞：

```
建站线 (Week 1-2):
  Phase 0 准备 → Phase 1 Hugo本地 → Phase 2 Waline
                                  → Phase 3 Cloudflare .com上线
                                  → Phase 5 CI/CD自动化
                                  → Phase 7 内容链路

备案线 (并行, 1-2周):
  Phase 4 ICP备案 ──────────────────────────────▶ Phase 6 阿里云CDN + .cn上线
```

**关键里程碑：**

| 里程碑 | 条件 | 意义 |
|--------|------|------|
| .com 上线 | Phase 3 完成 | 博客可对外访问，开始积累 Google 权重 |
| 点赞可用 | Phase 2 P0 验证通过 | 反馈收集渠道就绪 |
| 完整自动化 | Phase 5 完成 | push 即发布，双线同步 |
| .cn 上线 | ICP 备案完成 + Phase 6 | 国内首屏 < 1s，百度 SEO 生效 |

### 初始化项目

```bash
cd D:/01-laodao/blog
git init
git remote add origin https://github.com/laodao-ai/blog.git

# 在当前目录初始化 Hugo（目录已存在，用 --force）
hugo new site . --force

# 安装 PaperMod 主题（用 submodule 便于后续更新）
git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod themes/PaperMod
```

### hugo.toml 核心配置

```toml
baseURL = "https://laodao-ai.com"   # 备案完成后改为 .cn
title = "老刀AI码场"
theme = "PaperMod"
languageCode = "zh-CN"
defaultContentLanguage = "zh"

[params]
  ShowReadingTime = true
  ShowCodeCopyButtons = true     # P0: 代码一键复制
  ShowPostNavLinks = true
  ShowBreadCrumbs = true

[markup.highlight]
  noClasses = false
  lineNos = false                # 先关闭行号，避免与复制按钮的已知样式冲突

[outputs]
  home = ["HTML", "RSS"]
  section = ["HTML", "RSS"]

[outputFormats.RSS]
  mediaType = "application/rss+xml"
  baseName = "index"
```

## Phase 2 — Waline 部署

**时长**：~2h  
**可与 Phase 1 并行**  
**完成标志**：国内手机（非 VPN）能正常加载点赞按钮

### 步骤 1：LeanCloud 国际版

1. 注册 [LeanCloud 国际版](https://console.leancloud.app)
2. 创建应用，记录：
   - `AppID`
   - `AppKey`
   - `MasterKey`
   - API 域名（REST API 服务器地址）

### 步骤 2：Railway 部署 Waline

1. 进入 [Railway](https://railway.app)，用 GitHub 账号登录
2. 搜索 Waline 模板，一键部署
3. 配置环境变量：
   ```
   LEAN_ID=<AppID>
   LEAN_KEY=<AppKey>
   LEAN_MASTER_KEY=<MasterKey>
   ```
4. 记录部署后的域名，格式：`https://xxx.railway.app`

### 步骤 3：Hugo 接入 Waline

在 `hugo.toml` 中添加：
```toml
[params.waline]
  serverURL = "https://xxx.railway.app"
  reaction = true       # 开启点赞
  comment = false       # 关闭评论（国内监管）
  pageview = true       # 开启阅读量统计（仅后台可见）
```

### ⚠️ P0 验证：国内可达性测试

**必须在上线前完成，不可跳过。**

```
测试方法：
1. 用手机（关闭 VPN，使用移动/联通/电信流量）
2. 访问本地或测试环境的文章页
3. 检查点赞按钮是否正常显示和响应
4. 用 Chrome DevTools Network 面板确认 railway.app 请求无报错
```

**如果测试失败**（railway.app 被封），执行 Fallback：

```
Fallback 方案（按优先级）：
1. Vercel 部署 Waline + 绑定自定义域名
   → waline.laodao-ai.cn（备案后）或 waline.laodao-ai.com（备案前）
2. 阿里云函数计算部署 Waline（国内托管，最稳）
```

## Phase 5 — GitHub Actions 完整 CI/CD

**时长**：~2h  
**依赖**：Phase 3 完成  
**完成标志**：push main 分支自动触发双线部署

Cloudflare Pages 已有自动部署，这一步主要补全阿里云 OSS 推送。

### GitHub Secrets 配置

在 `laodao-ai/blog` repo → Settings → Secrets and variables → Actions：

```
ALIYUN_ACCESS_KEY_ID      阿里云 RAM 用户 AccessKey ID
ALIYUN_ACCESS_KEY_SECRET  阿里云 RAM 用户 AccessKey Secret
OSS_BUCKET                存储桶名称（如 laodao-ai-blog）
OSS_ENDPOINT              地域节点（如 oss-cn-hangzhou.aliyuncs.com）
```

### Workflow 文件

`.github/workflows/deploy.yml`：

```yaml
name: Deploy Blog

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: true
          fetch-depth: 0
      - uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: 'latest'
          extended: true
      - run: hugo --minify
      - uses: actions/upload-artifact@v4
        with:
          name: public
          path: public/

  deploy-cloudflare:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: public
          path: public/
      # Cloudflare Pages 已通过 Git 集成自动部署，此 job 可留空或用于额外验证

  deploy-aliyun:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: public
          path: public/
      - uses: manyuanrong/setup-ossutil@v2
        with:
          endpoint: ${{ secrets.OSS_ENDPOINT }}
          access-key-id: ${{ secrets.ALIYUN_ACCESS_KEY_ID }}
          access-key-secret: ${{ secrets.ALIYUN_ACCESS_KEY_SECRET }}
      - run: ossutil sync public/ oss://${{ secrets.OSS_BUCKET }}/ --delete
```

## Phase 6 — 阿里云 CDN + .cn 上线

**时长**：~2h  
**依赖**：ICP 备案完成（Phase 4）  
**完成标志**：`laodao-ai.cn` 国内首屏加载 < 1s

### 步骤

1. **阿里云 OSS**：
   - 创建 bucket（与 Phase 5 secrets 一致）
   - 开启静态网站托管，首页文档 `index.html`，404 文档 `404.html`
   - 设置 bucket 读权限为公共读

2. **阿里云 CDN**：
   - 添加加速域名 `laodao-ai.cn`
   - 源站类型选 OSS，绑定对应 bucket
   - 开启 HTTPS（申请免费证书）
   - 缓存策略：HTML `no-cache`，静态资源（CSS/JS/图片）`30天`

3. **DNS 配置**：
   - `laodao-ai.cn` CNAME 指向 CDN 域名（阿里云控制台会给出）

4. **更新 Hugo 配置**：
   ```toml
   baseURL = "https://laodao-ai.cn"
   # .com 通过 canonical 指向 .cn，避免 SEO 权重分散
   ```

5. **验证**：
   - [ ] 国内 Lighthouse Performance > 85
   - [ ] `laodao-ai.cn/index.xml` RSS 正常

### PicGo 图片工作流配置

```
安装 PicGo → 配置阿里云 OSS 插件：
  - Bucket: <OSS bucket 名称>
  - Area: <地域>
  - AccessKey / SecretKey: RAM 用户凭证
  - CustomUrl: https://img.laodao-ai.cn  ← 图片 CDN 域名

工作流: 截图 → 拖入 PicGo → 自动上传 → 返回 CDN URL → 粘贴进 Markdown
```

## 风险与关注点

| 风险 | 级别 | 处理方式 |
|------|------|----------|
| Waline railway.app 国内被封 | P0 | Phase 2 验证，失败立即切 Fallback |
| ICP 备案超时（>4周） | P1 | 不影响 .com 运营，延后 .cn 上线 |
| Hugo extended 版本安装失败 | P2 | 手动下载 release 包 |
| 阿里云 OSS 推送权限配置错误 | P2 | RAM 用户最小权限：`AliyunOSSFullAccess`（后续收紧） |
